用户问题

毕业论文写完后，排版
仍然是一项高成本任务

用户原有方案的不足

⏱ 手动使用 Word 排版 — 耗时费力

  • 全文逐段检查和修改格式
  • 修改内容后需重新确认格式
  • 页眉、页码、分节操作门槛高
  • 格式规范分散，容易遗漏

🔧 使用 LaTeX 模板 — 工具不适配

  • 本学院无官方 LaTeX 模板
  • 毕业论文以 Word 文档提交
  • 现有工具难以适配学院实际要求

目标用户

  • 正在完成毕业论文的高校学生
  • 使用 Word 撰写和提交论文的用户
  • 对复杂页眉页码和章节格式不熟悉的用户
  • 希望减少重复排版和检查时间的用户

核心能力

📄 自动排版 — 已支持模块

封面页边距 中英文摘要标题/正文样式 行距缩进目录 页眉页脚页码格式 参考文献致谢 原创性声明研究成果

⚙ 可配置规则

页面大小、页边距、页眉页脚距离、页码样式/起始值/字体、正文字体/字号/缩进/行距、三级标题样式等。配置保存为 YAML，便于复用和版本管理。

🔒 本地隐私保护

论文仅在本地读取处理，不上传服务器；源文件只读不覆盖；结果输出到新文件和任务目录。

版本迭代

三个版本的产品演进

V1 · 固定模板版本

验证可行性

验证"按学院规范自动生成可编辑 Word 毕业论文"的产品价值和技术可行性。

主要功能：封面、摘要、正文、标题、参考文献、声明和研究成果等模块基础排版。

暴露问题：格式值写死在程序中，模板更新需修改代码，无法保存不同配置。

▲ V1 固定模板版本（点击放大图片）

V2 · 配置驱动版本

适应变化

将格式规则从程序逻辑中拆分，提高对不同模板和规则变化的适应能力。

核心调整：YAML 配置、桌面界面改格式、多配置版本保存、模块化配置。

暴露问题：配置项增加后页面信息密集、按钮过多、导航层级不清。

▲ V2 配置驱动版本（点击放大图片）

V3 · 任务流程式界面

易于使用

将以功能模块为中心的界面，调整为以用户任务为中心的分步操作流程。

V3 主要调整

原问题
  • 顶部页签和模块同时存在
  • 页面展示过多字段
  • 主/辅操作缺少区分
  • 日志占据较大空间
解决方案
  • 左侧完整步骤导航
  • 中央仅处理当前任务
  • 右侧检查状态+辅助信息
  • 卡片化内容+较大留白

"体验用户反馈新版工具'步骤清晰'。"

V1 解决"能不能做" → V2 解决"能否适配变化" → V3 解决"用户是否容易使用"
